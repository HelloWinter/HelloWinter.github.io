1.1-框架

发表于 2017-01-07 |

iOS的绘图例程（routines）主要来源于UIKit和QuartzCore框架。 它们由现代化的Objective-C接口（来自UIKit）以及较早的基于C的函数和Core Foundation样式的对象（来自QuartzCore）组成。 这些项目可并排存在于代码中。 下面是一个示例，显示了一个UIKit方式的绘图操作，和一个Quartz方式的绘图操作：

// Draw a rounded rectangle in UIKit
 UIBezierPath *bezierPath = [UIBezierPath bezierPathWithRoundedRect:inset cornerRadius:12];
 [bezierPath stroke];

// Fill an ellipse in Quartz
CGContextFillEllipseInRect(context, rect);

QuartzCore框架通常被称为Quartz或Quartz 2D。 后者是苹果公司的原生2D渲染和抗锯齿API的官方名称。 它也是在苹果开发库，“Quartz 2D编程指南”的主要绘图参考使用的名称。

这个框架的范围比Quartz 2D名称更广泛。 QuartzCore框架实现了合成，渲染，以及许多其他的图形特性。 例如，QuartzCore APIs包括动画，平铺和Core Image过滤（即使Core Image确切来说是一个自有框架）。

对于iOS开发人员来说，Quartz通常以其内部实现名称Core Graphics更为人所知。 这本书在全文中使用了相同意思可以互换的Quartz和Core Graphics两个名称。 大多数基于C的API是以两个字母的CG前缀开始，（这是）源自Core Graphics（首字母）的名称。 从几何（CGRect和CGPoint）到对象（CGColorRef，CGImageRef和CGDataProviderRef），Quartz提供了庞大，丰富的绘图技术。

Quartz及其前身已经存在了很长时间。 甚至可追溯到上世纪80年代，在Diaplay PostScript为NeXTStep操作系统提供图形技术支持时。 Quartz已经使用了与现代PDF非常相似的内部成像模型。

在每个iOS版本更新中，（苹果）将绘图API进一步向Objective-C转移，简化，增强和优化程序绘图。 然而，Core Graphics的函数在日常绘图开发中仍然发挥重要作用。 虽然较新的例程（routines）取代了许多常见的(基于Core Graphics的)任务，但它们并没有取代它们。 （所以）在现阶段，准备好在日常工作中同时使用两个框架吧。

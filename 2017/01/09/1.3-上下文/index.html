1.3-上下文

发表于 2017-01-09 |

每个iOS绘图操作都从一个上下文开始。 从概念上讲，上下文非常类似于空白页面的纸张或空画布。 他们代表着绘画的iOS目的地。 它们包含有关绘图介质状态的所有信息 - 例如，画布是以某种方式旋转或变换的，可以在画布上绘制什么样的颜色，您可以在任何点绘制的细节的程度，等等。

在iOS中，您主要使用两种绘图上下文：位图上下文和PDF上下文。 Core Image框架提供第三种上下文类型，用于执行图像处理任务而不是绘图。

位图上下文

位图上下文本质上是矩形数据数组。 该数据的大小取决于每个图像元素（或“像素”）表示的颜色的种类。 如图1-4左图所示，设备RGB每像素使用三个或四个字节，这取决于位图是不透明（3字节）还是透明的（4字节）。

不透明位图忽略半透明值，优化存储。 半透明图像使用所谓的alpha值。 该值存储在与实际颜色或亮度信息不同的字节中。 它指的是每个像素的半透明度。 设备RGB的颜色信息以3字节存储，每个字节对应一个红色，绿色或蓝色值。

设备灰色图像，如图1-4右侧所示，每个像素使用1或2个字节。 对于每个像素存储一个亮度字节，并且可选地存储一个透明度字节。

PDF上下文

从开发人员的角度来看，PDF上下文的工作方式与位图上下文非常相似。 你使用相同的命令和函数绘制它们。 设置颜色和绘制形状和文本就像你在视图或图像中那样绘制。 但是，存在差异。

PDF上下文在其“后台存储”中包含向量数据，以与解决方案无关的方式描述图形。 位图上下文是光栅化的。 它们使用像素阵列来存储绘制到其中的数据。

PDF上下文也可能包含多个页面。 您可以建立一个边界矩形，指定每个PDF页面的默认大小和位置。 空矩形（CGRectZero）默认为标准A（字母）页。 这是8.5*11英寸，或612*792点。 （第2章讨论点和像素之间的差异。）

PDF图像内部存储为基于矢量的命令序列。 这提供了在位图图形中看不到的固有的与分辨率无关的特性。 苹果在其文档中写道，“PDF文件是与分辨率的大小无关的 - 它们绘制的大小可以无限增加或减少，而不牺牲图像细节。 位图图像的用户感知质量与用户所要看到的位图的分辨率息息相关。

你绘制到这些上下文就像你绘制到位图上下文一样。 差异主要在于它们的目标（文件和数据表示）以及当您开始新页面时。

除了位图和PDF上下文之外，您还可能遇到Core Image上下文。

Core Image上下文

Core Image框架可以帮助您极快地处理图像。 它提供了对图像源应用数字图像处理和计算机视觉的例程。 有了它，您可以将过滤器、链过滤器一起使用，实现特征检测（查找面部和眼睛），并分析图像以产生自动调整。

Core Image上下文是特定于将Core Image对象渲染为Quartz 2D和OpenGL的图像上下文。 通过板载图形处理单元（GPU）加速，Core Image上下文与Core Video像素缓冲区集成。 Core Image使用自己的颜色风格（CIColor）和图像（CIImage），它已针对Core Image的快速过滤和图像处理功能进行了优化。

4.2-类的便捷方法

发表于 2017-02-02

UIBezierPath的类方法构建矩形，椭圆，圆角矩形和弧，提供对公共路径样式元素的单一调用访问：

  • Rectangles - bezierPathWithRect:用于界面中的任何类型的视图填充或矩形元素。您可以使用此方法创建任何类型的矩形。
  • Ovalsandcircles - bezierPathWithOvalInRect:提供了一种用于构建任何类型的圆和椭圆的工具。
  • Rounded rectangles - bezierPathWithRoundedRect:cornerRadius:构建iOS设计师非常喜欢的圆角矩形。对于按钮，弹出警告框和许多其他视图元素来说这些路径是完美的。
  • Corner-controlled rounded rectangles - bezierPathWithRoundedRect:byRoundingCorners:cornerRadii:使您能够创建圆角矩形，但是可选择需要哪个角成为圆角。这对于创建从屏幕的一侧滑入的警报和面板特别有用，在这些情况下您通常不想四个角都是圆角。
  • Arcs - bezierPathWithArcCenter:radius:startAngle:endAngle:clockwise:从指定的起始角度和结束角度绘制弧线。使用这些弧添加视觉装饰，创建风格化的视觉分隔，并可使整个GUI看起来更加柔和。

这些Bezier路径类方法为您的绘图提供了一个基本的起点，如例4-1所示，此代码通过组合多个形状构建新的贝塞尔曲线路径。在每个步骤，它将形状追加到不断增长的路径上，当描边时，这个结果路径产生如图4-2所示的“人脸”。

如例4-1所示，Bezier路径实例本质上是可变的。您可以添加，也可通过追加新的形状和元素来延长路径，此外，路径不需要是连续的。图4-2中的路径由四个不同的，不相交的子形状组成，可以绘制为一个独立的单元。

Note

这是本书使用例子的第一部分，顾名思义，示例显示了在绘制时应用API调用的样本，代码清单提供了实用程序和有用的代码段，您可以将它们嵌入到您自己的项目中。

图4-2这个人脸包括一条单独的Bezier路径。它是通过将类提供的形状，即椭圆形（也称为圆形），圆弧和两个矩形附加到路径实例而构造的。

尽管如果你的路径太复杂，你会遇到性能问题，但这通常只发生在用户驱动的交互情况下。我在我的Draw应用程序中将图形表示为路径。就像是小孩子连续画画5-10分钟而不停歇一样，路径变得漫长会造成速度减慢。你如果使用普通的矢量绘图预建并嵌入自己应用程序的任何种类的（理想情况下）路径都不会遇到这个问题。

将描边宽度和填充或描边的颜色应用于所有元素，路径是一次绘制而成的，但是您将需要为逻辑单元拆分路径。当您需要添加多个绘图样式时，应该创建单独的路径，每个路径代表一个统一的操作。

图4-3带圆角的嵌套矩形。

仔细查看每个图像的角。内部路径的角半径在第一图像和第二图像之间并不同。左图像将其内部路径偏移4个点，并使用与第一个路径相同的12点角半径来绘制：

第二个执行相同的4点内偏移，但将拐角半径从12点减少到8.将插入匹配到拐角半径创建了两条路径之间更好的拟合，确保两条曲线从开始到结束都是平行，产生了如你在右边的图像看到的视觉上更加令人愉快的结果。
